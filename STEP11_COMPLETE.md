## 🎯 How to Deploy (Quick Version)

### 1. Prepare Supabase (30 minutes)

```bash
# 1. Create Supabase project at supabase.com
# 2. Run 3 SQL migrations in order
# 3. Copy URL and anon key to .env.local
# 4. Update auth redirect URLs (after Vercel deploy)
```

### 2. Test Locally (30 minutes)

```bash
npm install
npm run dev
# Test everything in TESTING_CHECKLIST.md
```

### 3. Deploy to Vercel (30 minutes)

```bash
# 1. Push to GitHub
git add .
git commit -m "Ready for production"
git push origin main

# 2. Import to Vercel
# - Go to vercel.com/new
# - Import repository
# - Add environment variables
# - Deploy!
```

### 4. Post-Deploy Setup (30 minutes)

```bash
# 1. Update Supabase auth URLs with Vercel domain
# 2. Test production deployment
# 3. Create first user (PSM role)
# 4. Invite team members
```

**Total Time: 2 hours to production!**
